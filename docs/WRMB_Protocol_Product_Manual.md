# WRMB 储蓄与收益协议 - 产品手册

## 1. 生态系统概述

WRMB 协议是一个创新的去中心化储蓄与收益协议。其核心并非一个传统的稳定币，而是一个通过价值累积模型为用户提供真实收益的系统。

整个生态系统由三部分协同工作：

1.  **`WRMB-protocol-contracts`:** 定义协议所有规则和机制的核心智能合约。
2.  **`wrmb-dapp`:** 功能最全的主 DApp，提供了与协议所有功能（储蓄、债券、封装）交互的界面。
3.  **`wrap-dapp`:** 一个轻量级的、单一功能的 DApp，专门用于 `WRMB` 和 `sRMB` 之间的快速兑换。

### 独特的代币模型

理解 WRMB 协议的关键在于理解其三种“代币”的角色：

*   **`WRMB` (可流通代币):**
    *   **是什么:** 一个标准的 ERC20 代币，可以在 Uniswap 等二级市场上自由交易。
    *   **作用:** 协议的**价值载体**和**流动性媒介**。用户从外部市场获得它，并最终将收益以它的形式体现。

*   **`sRMB` (储蓄凭证代币):**
    *   **是什么:** 一种受限制的、**不可自由转让**的 ERC20 代币。
    *   **作用:** 它是用户参与协议内部储蓄系统的**“入场券”和“出场券”**。用户只能通过官方的“封装”合约，用 `WRMB` 1:1 兑换它。它的存在确保了所有储蓄活动都必须通过协议的官方渠道进行。

*   **`sWRMB` (金库份额):**
    *   **是什么:** 这**不是一个代币**，而是用户在 `SavingsVault`（存款金库）中所占**份额**的内部记账单位。
    *   **作用:** 代表了用户在金库中的所有权。随着金库通过其投资策略（例如，通过 AMO 模块做市）不断产生利润，金库的总资产会增加，从而导致每一份 `sWRMB` 份额的内在价值（NAV）随之增长。

## 2. 核心用户旅程：储蓄与收益的“飞轮”

用户的完整收益周期如下：

**第一步：进入系统 (Wrap)**
1.  用户在二级市场（如 Uniswap）上购买获得流动的 `WRMB` 代币。
2.  用户访问 `wrmb-dapp` 的“封装”页面，或直接使用更简洁的 `wrap-dapp`。
3.  用户执行“封装”操作，将其持有的 `WRMB` 按 1:1 的比例兑换成受限制的 `sRMB` 代币。
    *   *例如：用户用 1000 `WRMB` 兑换得 1000 `sRMB`。*

**第二步：存入金库，开始赚取收益 (Deposit)**
1.  用户访问 `wrmb-dapp` 的“储蓄 (Savings)”页面。
2.  用户执行“存入”操作，将其持有的 `sRMB` 存入 `SavingsVault`（存款金库）。
3.  作为回报，金库根据当前的份额净值，给用户记上对应数量的 `sWRMB` 份额。
    *   *例如：用户存入 1000 `sRMB`，获得了 1000 `sWRMB` 份额。*

**第三步：价值累积**
*   用户无需任何操作。
*   `SavingsVault` 会利用其汇集的资金，通过各种策略（如 `v4-pool-amo` 模块）产生利润。
*   这些利润会使 `SavingsVault` 的总资产增加，从而使每一份 `sWRMB` 份额的价值不断增长。

**第四步：提取收益 (Withdraw)**
1.  经过一段时间后，用户决定提取收益。
2.  用户回到 `wrmb-dapp` 的“储蓄”页面。
3.  用户执行“提取”操作，将其持有的 `sWRMB` 份额兑换回 `sRMB` 代币。
4.  由于 `sWRMB` 份额的价值已经增长，用户现在会得到比当初存入时**更多**的 `sRMB` 代币。
    *   *例如：用户当初的 1000 `sWRMB` 份额现在价值增加了 10%，因此可以兑换回 1100 `sRMB`。这多出的 100 `sRMB` 就是用户的收益。*

**第五步：退出系统 (Unwrap)**
1.  用户回到“封装”页面或 `wrap-dapp`。
2.  用户执行“解封装”操作，将其持有的 `sRMB` 按 1:1 的比例兑换回可自由流通的 `WRMB` 代币。
    *   *例如：用户将 1100 `sRMB` 兑换成 1100 `WRMB`。*
3.  至此，用户可以将这 1100 `WRMB` 在二级市场上出售，完成收益的最终变现。

## 3. 其他功能

### 债券 (Bonds)

*   `wrmb-dapp` 还提供了一个“债券”功能。
*   这是一种协议融资机制。用户可以用其他资产（如 USDT）来购买协议发行的债券，并在未来以折扣价获得 `WRMB` 代币。
*   对于协议而言，这是其国库（Treasury）获取核心资产（如稳定币）的一种方式。
*   对于用户而言，这提供了一种折价获取 `WRMB` 的途径。
