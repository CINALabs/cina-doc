# 数字基金管理后台 - 产品操作手册

## 1. 概述

本应用是一个基于 Web 的内部管理仪表盘，专为“数字基金”的管理员设计。它提供了一套完整的工具，用于管理基金的日常运营、处理投融资请求以及维护系统的安全与合规。

**重要提示:** 该应用为内部工具，需要授权的管理员地址通过钱包连接后方可使用。其后端服务实现未知，本文档仅基于前端界面功能进行描述。

## 2. 核心功能模块

### 2.1. 仪表盘 (Dashboard)

后台首页，提供基金关键绩效指标（KPI）的概览。
*   **展示内容:**
    *   基金的最新资产净值 (NAV)。
    *   待处理的铸造（申购）和销毁（赎回）请求数量。
    *   基金的总锁仓价值 (TVL)。
    *   链上关键事件的日志。
*   **目的:** 帮助管理员快速了解基金的整体健康状况和运营负荷。

### 2.2. 资产净值管理 (NAV Management)

NAV 是衡量基金价值的核心指标，此模块用于其管理和验证。
*   **功能:**
    *   **提交 NAV:** 管理员定期（例如每日）通过此界面提交基金的最新 NAV 报告。这可能需要填写资产明细、总价值等信息。
    *   **NAV 验证:** 可能存在一个审批流程，需要另一位管理员对提交的 NAV 报告进行复核和“验证”。通过验证的 NAV 才会成为协议的官方标准。
    *   **历史记录:** 查看所有历史 NAV 提交和验证的记录。

### 2.3. 铸造管理 (Mint Management)

管理投资者的申购（铸造基金代币）请求。
*   **流程:** 投资者的申购请求不会自动在链上执行，而是首先作为一条“待处理”记录进入本管理后台。
*   **功能:**
    *   **请求列表:** 以列表形式展示所有待处理、已批准、已拒绝的铸造请求。
    *   **请求详情:** 点击可查看每个请求的详细信息，如投资者地址、申购金额、提交时间等。
    *   **审批操作:** 管理员可以对“待处理”的请求执行“批准”或“拒绝”操作。
    *   **执行铸造:** “批准”一个请求后，管理员可能需要通过此界面触发最终的链上交易，为投资者铸造相应的基金代币。

### 2.4. 销毁管理 (Burn Management)

管理投资者的赎回（销毁基金代币）请求，流程与铸造管理类似。
*   **流程:** 投资者的赎回请求同样需要管理员审批。
*   **功能:**
    *   **请求列表:** 展示所有待处理、已批准、已拒绝的销毁请求。
    *   **审批操作:** 管理员对请求进行“批准”或“拒绝”。
    *   **执行销毁:** “批准”后，管理员触发链上交易，销毁投资者的基金代币并将对应价值的底层资产退还给投资者。

### 2.5. 黑名单管理 (Blacklist Management)

用于管理地址访问权限，是基金合规与风险控制的重要工具。
*   **功能:**
    *   **添加地址到黑名单:** 管理员可以输入某个用户地址，并将其加入黑名单。被列入黑名单的地址将无法与基金的智能合约进行交互。
    *   **从黑名单移除地址:** 将地址从黑名单中移除，恢复其访问权限。
    *   **查看黑名单:** 展示当前所有被列入黑名单的地址列表。

### 2.6. 合约管理 (Contract Management)

提供对基金智能合约的直接管理功能。
*   **功能:**
    *   **暂停/恢复:** 在紧急情况下，管理员可以通过此界面暂停或恢复一个或多个核心合约的功能。
    *   **设置参数:** 修改智能合约的可调参数，例如管理费率、铸造/销毁费用等。
    *   **更新地址:** 更换合约依赖的其他关键地址，如预言机地址、资金管理员地址等。
